# 🛡️ Analyse de Risques SSI selon la Méthode ISO/IEC 27005
> **Gouvernance, Risque et Conformité (GRC) | Étude de cas secteur Distribution**

Ce projet simule une mission de conseil pour une chaîne nationale de distribution. L'objectif était de structurer une démarche de gestion des risques conforme aux standards internationaux pour protéger les actifs critiques de l'organisation.

## 📋 Démarche Méthodologique
L'analyse a été menée selon le cycle itératif de l'ISO 27005 :

1.  **Établissement du contexte** : Définition du périmètre (Siège, Magasins, E-commerce) et des critères d'acceptation des risques.
2.  **Identification des Actifs Primordiaux** : 
    *   Processus de vente et encaissement.
    *   Bases de données clients (RGPD).
    *   Chaîne logistique et approvisionnement.
3.  **Évaluation des Menaces** : Modélisation des scénarios redoutés (Ransomware, sabotage interne, erreur de manipulation, sinistre physique).
4.  **Estimation de la Criticité** : Utilisation d'une matrice Impact (1-4) x Vraisemblance (1-4).
    *   *Exemple* : Ransomware sur serveurs de caisse -> Impact : Majeur | Vraisemblance : Forte | **Risque : Critique**.

## 🛠️ Plan de Traitement des Risques (PTR)
Élaboration d'une feuille de route budgétisée incluant :
*   **Mesures de réduction** : Segmentation réseau IT/OT, authentification MFA, politique de sauvegarde déconnectée.
*   **Mesures de transfert** : Clauses contractuelles avec les prestataires cloud et cyber-assurance.
*   **Suivi** : Définition d'indicateurs (KPI) pour mesurer l'efficacité des mesures dans le temps.
